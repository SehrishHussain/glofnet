# Data Directory

This directory contains all datasets used throughout the GLOFNet reproduction project.

The data is organized according to its role in the pipeline to ensure reproducibility, maintainability, and a clear separation between downloaded, generated, and reference datasets.

---

## Directory Structure

```
data/
│
├── README.md
├── raw/
├── interim/
├── processed/
└── reference/
```

---

# raw/

Contains datasets downloaded directly from external sources.

These files represent the original inputs to the pipeline and should remain unchanged after download.

```
raw/
│
├── sentinel/
├── itslive/
└── modis/
```

### Sentinel

Contains raw Sentinel-2 imagery downloaded from Google Earth Engine.

Examples

- GeoTIFFs
- Exported image tiles

---

### ITS_LIVE

Contains raw glacier velocity products downloaded from the NASA ITS_LIVE dataset.

Examples

- NetCDF files
- Velocity rasters

---

### MODIS

Contains raw MODIS Land Surface Temperature products.

Examples

- GeoTIFFs
- NetCDF files

---

# interim/

Contains temporary outputs generated during preprocessing.

These files are intermediate products used between pipeline stages.

Examples

- Cloud masked imagery
- Reprojected rasters
- Resampled datasets
- Temporary merged files

Files in this directory can be regenerated at any time.

---

# processed/

Contains the final cleaned datasets produced by the preprocessing pipeline.

Examples

- Normalized Sentinel patches
- Daily glacier velocity time series
- Temperature anomaly sequences
- Harmonized multimodal dataset

These datasets are used for machine learning and downstream analysis.

---

# reference/

Contains static datasets that define the study area or provide supporting geographic information.

Unlike the raw datasets, these files are not downloaded each time the pipeline runs.

```
reference/
│
├── RGI/
├── boundaries/
├── DEM/
└── ...
```

Examples include

- Randolph Glacier Inventory (RGI)
- Karakoram Glacier Inventory (KGI)
- Glacier outlines
- Administrative boundaries
- Digital Elevation Models (DEM)

These datasets act as reference layers for the pipeline.

# Reproducibility

The objective of this repository is to reproduce the preprocessing pipeline described in:

> GLOFNet: A Multimodal Dataset for GLOF Monitoring and Prediction

All datasets used are publicly available.

No manually modified data should be stored in this directory unless explicitly documented.

---

# Version Control

Large datasets should **not** be committed to Git.

Instead, commit only:

- scripts
- metadata
- configuration files
- documentation

Downloaded datasets should be regenerated by running the pipeline whenever possible.
